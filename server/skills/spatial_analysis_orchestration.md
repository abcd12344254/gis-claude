## 空间分析编排流程

### 核心原则

1. **分析前必须先有数据** — 无图层时先生成 OSM 查询指令
2. **坐标系前置检查** — 面积/距离/缓冲区/热点/插值分析前强制 CRS 检查
3. **引用图层用核心关键词** — 系统自动模糊匹配

### 坐标系处理规则

#### 强制转换场景

| 分析类型 | 处理 |
|----------|------|
| 面积计算 | **必须**转换为等面积投影 |
| 距离测量 | **必须**转换为等距离投影或 UTM |
| 缓冲区分析 | **必须**转换为投影坐标系（缓冲距离单位需为米） |
| 热点分析 | **必须**转换为投影坐标系 |
| 网络分析 | **必须**转换为投影坐标系 |
| 空间插值 | **必须**转换为投影坐标系 |
| 格网生成 | **必须**转换为投影坐标系 |
| 点密度分析 | **必须**转换为投影坐标系 |

#### 检查流程

1. 调用 `get_data_info` 获取所有输入数据的 CRS
2. 若 CRS 为地理坐标系(单位:度)且分析类型需要投影坐标系 → 调用 `transform_crs`
3. 多图层操作 → 检查所有图层 CRS 一致，不一致则统一转换
4. 中国区域推荐：小范围(<6°经度) → UTM分带；省级 → CGCS2000 3度带；全国 → Albers 等面积

### 编排流程

#### 缓冲区分析
```
1. 检查 CRS → 需要投影坐标系
2. 校验缓冲距离单位与 CRS 单位一致
3. 执行 buffer_analysis
4. 验证结果有效性
```

#### 叠加分析
```
1. 检查所有图层 CRS 一致性
2. 检查空间范围是否有交集
3. 处理字段名冲突（自动重命名）
4. 执行 overlay_analysis
5. 处理可能的碎屑多边形
```

#### 热点分析
```
1. 确认点要素数据，检查点数量(>30)
2. 检查 CRS → 需要投影坐标系
3. 确定带宽参数（自动计算或用户指定）
4. 执行 hotspot_analysis (Getis-Ord Gi*)
5. 解释结果（Gi*指数、P值、热点/冷点分类）
```

#### 适宜性评价
```
1. 统一所有因子图层 CRS
2. 标准化各因子数据（归一化 0-100）
3. 校验权重和为 1（自动归一化）
4. 执行加权叠加 (suitability_analysis)
5. 结果分级输出（高/中/低适宜）
```

#### 空间插值
```
1. 确认点要素数据，检查点数量(>20)
2. 检查插值字段数据类型（数值型）
3. 检查 CRS → 需要投影坐标系
4. 基于数据特征推荐插值方法（IDW/Kriging/Spline）
5. 自动计算栅格分辨率（基于点密度）
6. 执行插值并交叉验证
```

### 错误处理

| 错误 | 处理方式 |
|------|----------|
| CRS 不匹配 | 自动转换为统一坐标系 |
| 几何无效 | 调用 fix_invalid_geometry 修复 |
| 字段缺失 | 提示用户选择替代字段 |
| 参数不合理 | 自动调整或提示确认 |
| 空间范围无交集 | 预警并询问是否继续 |

### 输出规范

每次分析完成后应输出：
1. 分析结果图层（自动加载到地图）
2. 关键统计信息（要素数、面积、距离等）
3. 所执行的预处理步骤说明（含坐标系转换记录）
4. CRS 信息（分析使用的坐标系）
