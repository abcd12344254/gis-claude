## 坐标系"隐形陷阱"检查清单

坐标系问题是 GIS 中最常见、最隐蔽的错误来源。
在每次空间分析前，必须自动检查以下清单：

### 强制检查清单

| 检查项 | 检查方法 | 处理策略 |
|--------|----------|----------|
| 数据是否包含 CRS 信息 | `get_data_info` → `crs` 字段 | 缺失则提示用户确认或根据坐标范围推断 |
| CRS 是否为地理坐标系 | `crs.is_geographic` | 面积/距离/缓冲区分析需转换 |
| 多图层 CRS 是否一致 | 比较各图层 `crs` 字段 | 不一致则统一转换到主图层 CRS |
| 分析类型与 CRS 单位是否匹配 | 分析类型 + CRS 单位对照 | 单位不匹配（度 vs 米）需转换 |
| 缓冲区距离单位与 CRS 单位是否一致 | 缓冲距离单位 vs CRS 单位 | 不一致需转换 CRS 或转换距离单位 |
| 栅格与矢量 CRS 是否一致 | 比较栅格和矢量 `crs` | 不一致需重投影 |

### 常见陷阱类型

| 陷阱类型 | 问题描述 | 错误后果 | 触发条件 |
|----------|----------|----------|----------|
| **经纬度直接计算面积** | 将 WGS84 经纬度坐标直接用于面积计算 | 面积值严重偏小（高纬度误差可达 50%+） | 用户要求计算面积，数据为 EPSG:4326 |
| **经纬度直接计算距离** | 将经纬度坐标当作平面坐标计算距离 | 距离值无实际意义 | 用户要求距离测量，数据为 EPSG:4326 |
| **不同坐标系数据叠加** | 多源数据坐标系不一致直接叠加 | 空间位置偏移、分析失败 | 多图层操作时各图层 CRS 不同 |
| **投影变形未考虑** | 未根据分析目的选择合适投影 | 面积/距离存在系统性偏差 | 跨越大范围的分析 |
| **缓冲区距离单位错误** | 经纬度坐标系下缓冲区距离单位为度 | 缓冲范围严重偏大或偏小 | 缓冲区分析时数据为地理坐标系 |

### 中国常用坐标系转换

| 原坐标系 | 目标坐标系 | 适用场景 | EPSG |
|----------|------------|----------|------|
| WGS84 (EPSG:4326) | CGCS2000 3度带 | 中国区域面积/距离 | EPSG:4549 等 |
| WGS84 (EPSG:4326) | UTM Zone 50N | 中国东部 (114°-120°) | EPSG:32650 |
| WGS84 (EPSG:4326) | UTM Zone 51N | 中国中部 (108°-114°) | EPSG:32651 |
| WGS84 (EPSG:4326) | Web 墨卡托 | Web 地图展示 | EPSG:3857 |
| GCJ-02/BD-09 | WGS84 | 国内GPS数据融合 | 需要 coord-transform |

### Agent 处理规范

执行以下分析类型时，必须先调用 `/api/crs/check`：

- `buffer` / `area` / `distance` / `hotspot` / `interpolation` / `network` / `grid` / `density` / `cluster` / `zonal`

若检查不通过（`passed: false`）：
1. 向用户说明 CRS 问题（展示 `report` 字段）
2. 自动执行 `transform_crs`（若 `suggested_crs` 可用）
3. 标注转换动作到分析结果
